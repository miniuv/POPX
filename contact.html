Reporting Bugs

If you encounter a bug or unexpected behavior in any POPX operator, please send a detailed report including:

Required Information

  • Operator name and version — Check the operator info panel in TouchDesigner
  • TouchDesigner build number — Found in Help → About TouchDesigner
  • Steps to reproduce the issue — Detailed description of how to trigger the bug
  • Screenshots or error logs if possible — Visual evidence or textport output

Example Bug Report

Subject: [Bug] Noise Modifier crashes with high iteration count

Operator: Noise Modifier v1.2.3
TouchDesigner Build: 2023.11760

Steps to reproduce:
1. Create a Scatter operator with 10000 points
2. Connect to Noise Modifier
3. Set Octaves parameter to 10
4. Set Frequency to 5.0
5. TouchDesigner freezes/crashes

Expected: Operator should process without crashing
Actual: Application becomes unresponsive

Screenshots attached showing parameter settings.

Response Time

We prioritize bug reports based on severity:

Priority Description Response Time
Critical Crashes, data loss, or complete operator failure 24 hours
High Major functionality broken, no workaround available 48 hours
Medium Feature not working as expected, workaround exists 3-5 days
Low Minor issues, cosmetic problems, feature requests 1-2 weeks
