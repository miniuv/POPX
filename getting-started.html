Getting Started with POPX

POPX Overview

POPX is a new family of operators that extends TouchDesigner's POPs (Point Operators). Designed to fully leverage GPU acceleration, POPX introduces a modular framework for generating, transforming, and simulating geometry in real time.

At its core, POPX focuses on instance transformation — enabling artists and developers to create, manipulate, and animate large-scale instanced geometry efficiently. The system revolves around POPX Geometry, a packed data structure where each piece of geometry is stored in a single point containing all necessary attributes for transformation, including position, rotation, scale, and pivot. This packed approach allows POPX to manage transformations directly on the GPU without requiring users to manually construct transform matrices or compute attribute operations.

Although POPX operators can generate and operate on POPX Geometry, they are also fully compatible with any standard POP geometry. This flexibility allows the POPX system to integrate smoothly into existing workflows while maintaining GPU efficiency and procedural control.

Built for artists, technical directors, and creative coders, POPX provides a unified environment for creating procedural animations, real-time simulations, generative visuals, and large-scale geometry transformations — all with precision, performance, and creative flexibility inside TouchDesigner.

Operator Types Overview

POPX operators are organized into five categories, each serving a specific role in the instance manipulation workflow:

Generators

Generators create and initialize new instances or geometry from various sources. They are the starting point of any POPX workflow, defining initial positions, attributes, and distribution patterns. Generators can emit from surfaces, volumes, curves, vertices, or procedural patterns, with full control over particle density, initial velocity, orientation, color, and custom attributes. They support both continuous emission over time and single-burst generation modes.

Falloffs

Falloffs define areas of influence or weighting for other operators. They generate spatial weight values (typically 0-1) based on distance from points, curves, surfaces, or volumes. These weight values can then modulate the strength of any modifier, allowing you to create localized transformations, forces, and deformations. Falloffs support custom remap curves, inversion, clamping, and can write their values to custom particle attributes for advanced control.

Modifiers

Modifiers transform or adjust existing instance attributes like position, rotation, scale, velocity, color, and more. They read particle data streams, apply transformations or effects, and output the modified stream for downstream processing. All modifiers support falloff integration for spatial control, can target specific particle groups, and offer blend modes for combining effects. Modifiers are fully GPU-accelerated and can be chained together to create complex procedural behaviors.

Tools

Tools perform utility functions, conversions, and data operations within the POPX workflow. Unlike modifiers, they don't directly transform particle motion but instead manage attributes, analyze data, prepare instances for rendering, and convert between formats. Tools include attribute managers for creating and renaming attributes, voxelizers for converting particles to volumes, instance setup operators for GPU rendering preparation, and analysis tools for debugging and performance optimization.

Simulations

Simulations handle time-based dynamics, interactions, and physics-like behaviors. They use industry-standard algorithms like FLIP (Fluid Implicit Particle) and SPH (Smoothed Particle Hydrodynamics) to calculate realistic particle motion based on forces, collisions, constraints, and fluid dynamics. Simulations support accurate time-stepping with sub-frame iterations, fast spatial hashing for collision detection, constraint solving with position-based dynamics, and are fully GPU-accelerated for real-time performance.
