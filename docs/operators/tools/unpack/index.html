Unpack

Summary

Unpack converts POPX Geometry into POP geometry by unpacking the packed points structure and extracting the underlying geometry. This operator reveals the actual mesh or primitive data that was stored inside the packed primitives.

Unlike POPX to which strips packed structure and returns clean points, Unpack extracts the full geometry contained within each packed primitive, giving you access to the vertices, faces, and primitive structure of the instanced objects. This is useful when you need to work with the actual geometry rather than just point representations.

You can optionally apply the packed primitive's transform to the unpacked geometry, and selectively transfer point attributes and groups from the packed points to the unpacked geometry. This gives you control over what data is preserved during the unpacking process.

Parameters

Page: Unpack

Apply Transform Applytransform – When enabled, applies the packed primitive's transform matrix to the unpacked geometry.
Transfer Attributes Transferattrs – Specifies which point attributes to transfer from the packed points to the unpacked geometry.
Transfer Groups Transfergroups – Specifies which point groups to transfer from the packed points to the unpacked geometry.

Page: Common

Free Extra GPU Memory Freeextragpumem – Free memory that has accumulated when output memory has grown and shrunk.

Inputs

Input 0 – POP – POPX Geometry with packed primitives to unpack

Outputs

Output 0 – POP – Unpacked POP geometry with extracted mesh/primitive data
