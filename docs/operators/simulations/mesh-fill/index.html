Mesh Fill

Summary

Mesh Fill is a GPU-accelerated simulation that fills mesh volumes or surfaces with particles using a density-guided algorithm. This operator drives points to move inside mesh geometry or volumetric fields, gradually filling the space with organic growth patterns. The simulation creates natural space-filling behavior by guiding particles through density fields generated from input geometry.

The operator supports both mesh-based and volume-based inputs with automatic voxelization for mesh geometry. You can control whether particles fill the entire interior volume or only the surface boundary, with adjustable radius controlling how closely particles pack together. Filtering parameters allow smoothing of the density field for more uniform or organic filling patterns. Optional seeding from density allows controlled spawning of new particles during simulation.

Mesh Fill outputs the filled point cloud and optionally generates trail geometry showing particle movement paths. The simulation includes standard playback controls (initialize, start, play) for controlling fill progression. This makes Mesh Fill ideal for creating organic growth animations, volumetric filling effects, space colonization patterns, or particle-based mesh representations.

Parameters

Page: Mesh Fill

Time Step Timestep – Controls simulation speed per frame (higher values = faster filling).
Switch to Volume Input Switchtovolumeinput – Use 3D texture input instead of mesh geometry for density field.
Normalize Input Mesh Normalizeinputmesh – Scale input mesh to unit size before voxelization.
Max Axis Resolution Maxaxisresolution – Voxel grid resolution for mesh-to-volume conversion (higher = more detail).
Precision Precision – Texture precision for density field (16-bit or 32-bit float).
Fill Surface Only Fillsurface – Restricts particles to surface boundary instead of filling entire volume.
Radius Radius – Particle spacing radius controlling packing density (smaller = tighter packing).
Filter Scale Filterscale – Multiplier for density field filtering iterations.
Filter Size Size – Blur kernel size for density field smoothing.
Initialize Initializepulse – Pulse to reset simulation and prepare density field.
Start Startpulse – Pulse to begin filling simulation from initialized state.
Play Play – Toggle continuous simulation playback.

Page: Seed

Spawn From Density Spawn – Enables spawning new particles from density field during simulation.
Seed Count Seedcount – Number of seed particles to spawn per frame when spawning enabled.
Seed Seed – Random seed for particle spawn positions.
Max Attempts Maxattempts – Maximum spawn position attempts before giving up (prevents infinite loops).

Page: Trails

Active Activetrails – Enables trail geometry generation showing particle movement paths.
Always Cook Alwayscook – Forces trail geometry to update every frame regardless of changes.
Length Length – Maximum trail length in frames (history buffer size).
Filter Type Filtertype – Smoothing filter for trail curve (gaussian, catmull-rom, etc.).
Edge Distance Filterdist – Distance metric for edge filtering along trail curves.
Effect Effect – Smoothing effect strength (0 = no smoothing, 1 = full effect).
End Points Fixed Endpointsfixed – Prevents smoothing from moving trail endpoints.

Page: Common

Free Extra GPU Memory Freeextragpumem – Free memory that has accumulated when output memory has grown and shrunk.

Inputs

Input 0 – Mesh or Volume – Mesh geometry or 3D texture volume defining fill space
Input 1 – Initial Particles (Optional) – Optional starting particles to seed the fill simulation

Outputs

Output 0 – Particles – Filled point cloud with particles distributed through mesh volume or surface
Output 1 – Trails (Optional) – Curve geometry showing particle movement paths (when Active enabled)
