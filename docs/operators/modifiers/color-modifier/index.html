Color Modifier

Summary

The Color Modifier modifies the color of each instance based on falloff values. It uses a color ramp (gradient) to map falloff attribute values to colors, enabling visually dynamic effects based on spatial relationships, proximity, or any custom falloff attribute.

You can customize the color palette using the Ramp parameter, which accepts either an internal ramp editor or an external Ramp TOP reference. This allows for flexible color mapping from smooth gradients to complex multi-color patterns.

Parameters

Page: Color

Group Group – If there are input groups, specifying a group name in this field will cause this POPX to act only upon the group specified.
Falloff Attribute Falloffattr – Specifies which attribute to use for falloff values that drive the color ramp lookup.
Open Ramp Openramp – Opens the internal ramp editor for customizing the color gradient.
Reset Ramp Resetramp – Resets the internal ramp to default gradient values.
Ramp TOP Ramptop – Reference to an external Ramp TOP for color palette. When specified, overrides the internal ramp editor.

Page: Common

Bypass Bypass – Pass through the first input to the output unchanged.
Free Extra GPU Memory Freeextragpumem – Free memory that has accumulated when output memory has grown and shrunk.
Render Primitives Renderprimitives – Toggles rendering of POPX Geometry or shows it as point instances only.
SRT / RST Srtrst – Sets the transform order when using POPX Geometry as built-in TouchDesigner instances.

Inputs

Input 0 POP – POPX/POP Geometry

Outputs

Output 0 POP – POPX_out1
