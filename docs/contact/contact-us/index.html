Reporting Bugs

If you encounter a bug or unexpected behavior in any POPX operator, please send a detailed report including:

Required Information

  • •Operator name and version — Check the operator About parameter page for version
  • •TouchDesigner build number — Found in Help → About TouchDesigner
  • •Steps to reproduce the issue — Detailed description of how to trigger the bug
  • •Screenshots or error logs if possible — Visual evidence or textport output

Response Time

We prioritize bug reports based on severity:

Critical
Crashes, data loss, or complete operator failure
24 hours
High
Major functionality broken, no workaround available
48 hours
Medium
Feature not working as expected, workaround exists
3-5 days
Low
Minor issues, cosmetic problems, feature requests
1-2 weeks
