Installation & Getting Started

The POPX toolkit is distributed as a collection of .tox components that can be imported into any TouchDesigner project. Each operator is self-contained and can be used independently or chained together to create complex particle systems and procedural effects.

Installation Steps

  1. Download or clone the POPX repository from GitHub, Gumroad, or Patreon.
  2. Extract the downloaded archive and locate the /popx directory containing all operators.
  3. In TouchDesigner, drag and drop the desired .tox file (e.g., POPX_Transform_Modifier.tox) directly into your network editor.
  4. Each operator comes pre-configured with standard POPX attributes and can be immediately chained with other operators.
  5. For updates, simply replace the component in your project with the latest version from the repository.

System Requirements

  • TouchDesigner: 2023.11 or later (Experimental builds recommended for latest features)
  • GPU: OpenGL 4.6+ with compute shader support
  • Operating System: Windows 10/11 or macOS 13+
  • RAM: 8GB minimum, 16GB+ recommended for large particle counts
  • GPU VRAM: 4GB minimum, 8GB+ recommended for simulations

Quick Start Guide

1. Import Your First Operator

Drag a Generator operator into your network to create initial particle data. Try POPX_Scatter_Generator.tox to start with a basic point cloud.

2. Add Modifiers

Chain Modifier operators downstream to transform and manipulate particles. Use POPX_Transform_Modifier.tox to move and rotate your particles.

3. Apply Falloffs

Add Falloff operators to create spatial constraints and weight masks for localized effects and smooth transitions.

4. Visualize & Export

Use Tool operators to visualize particle attributes, debug your network, and export data to other TouchDesigner components.

Best Practices

  • Start Simple: Begin with a single Generator and Modifier to understand the POPX workflow before building complex chains.
  • Use CHOPs: Drive parameters with CHOP channels for dynamic, time-based animations.
  • Attribute Awareness: POPX operators rely on standard attributes like P (position), v (velocity), and popxOrient (orientation).
  • GPU Optimization: POPX operators are GPU-accelerated. Monitor your GPU usage and adjust particle counts accordingly.
  • Version Control: Keep your POPX components organized and update them regularly from the official repository.

Troubleshooting

Operator Not Loading

Ensure you're using TouchDesigner 2023.11 or later. Check the Textport for error messages and verify GPU compute shader support.

Poor Performance

Reduce particle counts, disable unused modifiers, or lower resolution on texture-based operations. Monitor GPU usage in Task Manager or Activity Monitor.

Attributes Missing

Verify that upstream operators are outputting the required attributes. Use POPX Tool operators to inspect attribute data.
