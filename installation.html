Installation

Download POPX

POPX is available exclusively through Patreon. To access it, subscribe to the POPX Tier on my Patreon page and download the latest release.

Latest Version: v1.0.0
Release Date: November 2025

Each release includes the POPX operators package and an example project file to help you get started quickly. A full installation walkthrough is also available in the video tutorial.

System Requirements

TouchDesigner: 2025 Experimental (2025.xxx) or newer — required to access the new POPs (Point Operators) family.

GPU: NVIDIA or AMD GPU with support for Vulkan 1.1 / OpenGL 4.5.

Operating System:

  • Windows 10/11 — full support for all GPU features.
  • macOS Monterey (12) or newer — functional, but with limited GPU feature support.

GPU Memory: Minimum 4 GB VRAM, 8 GB+ recommended for large simulations.

Note: Some POPX operators use GPU atomic operations, which may not be fully supported on macOS or integrated GPUs.

Installation Steps

  1. Subscribe to the POPX Tier on Patreon and download the POPX.tox package.
  2. Open your TouchDesigner project.
  3. Drag the POPX.tox file directly into your network.
  4. Your project may freeze momentarily while the system initializes — this is normal.
  5. Once loaded, the POPX container will appear at the root level of your project.
  6. Press Tab to open the OP Create Dialog — you'll now see the new POPX operator family available for use.

Verify Installation

To verify that POPX is installed correctly:

  1. Create a new TouchDesigner project
  2. Add a POPX Scatter operator to your network
  3. Connect a Box SOP to the first input of POPX Scatter
  4. Create a Geometry COMP and connect POPX Scatter to it
  5. You should see particles scattered on the box surface

If particles appear, your installation is successful!

  2. Scatter particles: Add POPX Scatter, set "Total Count" to 1000

  4. Render: Create Geometry COMP, enable "Instancing"

Troubleshooting

Operators Not Appearing in Palette

  • Verify the palette path in Edit → Preferences → Palette
  • Restart TouchDesigner after adding the palette folder
  • Check that .tox files are not corrupted (re-download if necessary)

GPU Compute Errors

  • Update your graphics drivers to the latest version
  • Verify OpenGL 4.6 support with glxinfo (Linux) or GPU-Z (Windows)
  • Reduce particle count if experiencing out-of-memory errors

Performance Issues

  • Enable "Cook on Demand" for operators not actively being used
  • Reduce particle counts during initial setup, scale up for final renders
  • Use falloff operators to limit modifier influence regions
  • Monitor VRAM usage in TouchDesigner's performance monitor
