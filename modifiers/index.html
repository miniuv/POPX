POPX Modifiers

Transform, deform, and manipulate particle streams with forces, noise, constraints, and custom expressions.

Overview

Modifiers are the core manipulation tools in the POPX ecosystem. They read particle data streams, modify attributes (position, velocity, orientation, scale, color, etc.), and output the transformed stream for downstream processing or rendering.

All modifiers support:

  • Falloff Integration – Connect falloff operators to spatially limit modifier influence
  • Custom Attributes – Modify any particle attribute, including user-defined attributes
  • GPU Acceleration – All operations run on compute shaders for real-time performance
  • Chainable Operations – Combine multiple modifiers to create complex procedural systems

Available Modifiers

Transform Modifier

Core

Applies translation, rotation, and scaling transformations to particles and instances with support for delta transforms and orientation accumulation.

Noise Modifier

Core

Deforms particle positions and attributes using Perlin, Simplex, Curl, and Voronoi noise patterns with customizable frequency and amplitude.

Force Modifier

Physics

Applies directional forces, gravity, turbulence, drag, and vortex forces to particle velocities with real-time physics integration.

Color Modifier

Visual

Modifies particle color attributes using ramps, gradients, noise, and attribute-driven coloring with HDR support.

Velocity Modifier

Physics

Controls particle velocity through direct manipulation, speed limits, alignment forces, and velocity inheritance from geometry.

Attribute Transfer

Data

Transfers attributes from source geometry to particles using nearest point, proximity averaging, or texture sampling methods.

Constraint Modifier

Physics

Constrains particles to surfaces, volumes, curves, and bounding regions with collision response and sliding behavior.

Expression Modifier

Advanced

Create custom particle behaviors using GLSL expressions with access to all attributes, neighbors, and global parameters.

Modifier Workflow

A typical modifier workflow involves chaining operators to build complex behaviors: 

POPX Scatter (create particles)
  ↓
POPX Force Modifier (apply gravity + turbulence)
  ↓
POPX Noise Modifier (add position displacement)
  ↓
POPX Transform Modifier (add rotation)
  ↓
POPX Color Modifier (color by velocity)
  ↓
Geometry COMP (render)

Best Practices

  • Order Matters – Physics modifiers (Force, Velocity) should generally come before visual modifiers (Color, Transform)
  • Use Falloffs – Connect falloff operators to limit modifier influence to specific regions
  • Enable/Disable – Use the "Enable" parameter to toggle modifiers without breaking the chain
  • Attribute Names – Stick to standard attribute names (P, v, Cd, N, orient) for compatibility with other operators

Common Parameters

Most modifiers share these common parameters:

Parameter Type Description
Enable Toggle Turns the modifier on/off (passthrough when disabled)
Strength Float Global multiplier for modifier effect (0-1 range typically)
Falloff Input SOP Optional falloff field to spatially modulate strength
Group String Limit modifier to particles in specified group
Blend Mode Menu How modifier combines with existing attributes (Add, Multiply, Replace)
